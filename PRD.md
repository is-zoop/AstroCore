# AstroCore 数据分析平台 PRD

## 1. 文档信息

- 产品名称：AstroCore  数据分析平台
- 文档类型：产品需求文档
- 版本：V0.1
- 编写依据：根据 `templates` 前端界面功能反推
- 实现方案：后端使用FastAPI实现，第一版主要实现现在已有的功能，尽量不调整现有的前端布局

## 2. 产品概述

AstroCore 是一个面向企业内部的数据分析与可视化看板平台。平台提供安全登录、看板浏览、数据筛选、数据集管理、数据源连接、用户权限管理和系统品牌配置能力，帮助业务人员快速查看数据分析结果，帮助管理员集中维护看板、数据资产和访问权限。

当前前端界面已体现两类使用场景：

- 普通用户：登录系统、选择并查看可访问的数据看板、调整查询条件、维护个人资料和登录密码。
- 管理员：维护看板、数据集、数据连接、用户账号和系统显示配置。

## 3. 产品目标

- 建立统一的数据看板访问入口，降低业务人员查看分析结果的成本。
- 将看板、数据集、数据源和用户权限纳入后台统一管理。
- 支持按角色区分功能入口，保障数据资产和系统配置的访问安全。
- 提供可配置的系统名称、图标和 Logo，满足内部平台品牌化需求。

## 4. 用户角色

### 4.1 管理员

管理员拥有完整后台管理权限，可访问：

- 我的看板
- 看板管理
- 数据集管理
- 数据连接
- 用户管理
- 系统设置
- 个人设置

### 4.2 普通用户 / 查看者

普通用户主要用于查看数据看板，可访问：

- 我的看板
- 个人设置
- 退出登录

后台管理类功能对普通用户不可见或不可访问。

## 5. 功能范围

### 5.1 登录认证

用户进入系统时，若未登录，应跳转至登录页。

登录页应提供：

- 用户名输入
- 密码输入
- 登录提交
- 加载状态
- 登录失败提示
- 登录成功后进入主工作台

登录成功后，系统应保存当前用户会话状态，并记录用户基础信息，包括：

- 用户 ID
- 用户名
- 角色

退出登录后，应清理会话状态并返回登录页, 用户没有操作超过60分钟后，需要重新登录。

### 5.2 主工作台布局

登录后进入主工作台，整体布局包括：

- 左侧导航栏
- 顶部标题与全局操作区
- 主内容展示区
- 底部用户信息与用户菜单

左侧导航栏应支持展开和收起。收起后保留图标入口，展开后显示完整菜单名称。

顶部区域应包含：

- 当前页面标题
- 当前选中的看板名称
- 全局搜索输入框
- 通知入口

用户菜单应包含：

- 个人设置
- 退出系统

### 5.3 我的看板

用户可在左侧“我的看板”菜单中展开可访问看板列表。

看板列表项应展示：

- 看板名称
- 看板分类
- 看板图标

用户选择某个看板后，主内容区应展示该看板页面。

看板页面应包含查询条件区：

- 日期范围
- 筛选条件
- 运行查询按钮

看板内容区用于承载后续的数据可视化画布 ，需要预留看板显示入口，由后续在看板管理中发布看板后在我的看板中显示；

### 5.4 看板管理

管理员可进入看板管理页面，对系统内看板进行维护。

页面应支持：

- 搜索看板
- 新建看板
- 查看看板列表
- 编辑看板
- 删除看板
- 修改看板图标

看板列表应展示：

- 看板名称
- 绑定数据集
- 发布状态
- 所属人
- 最近编辑时间
- 看板图标

发布状态包括：

- 已发布
- 草稿
- 下架

编辑看板：

- 可以编辑看板名称、绑定数据集、发布状态
- 可以上传看板文件,上传的文件单独保存到看板文件夹中，文件夹使用看板ID命名
- 看板文件（一般也是html文件，由streamlith构建)上传后同于显示在我的看板中
- 仅发布状态为已发布的看板可以在我的看板中查看

修改看板图标时，应提供图标选择弹窗，管理员可选择预置图标并确认保存。

### 5.5 数据集管理

管理员可进入数据集管理页面，维护用于看板展示的数据集。

页面应支持：

- 搜索数据集
- 创建数据集
- 查看数据集列表
- 编辑数据集 SQL 或查询逻辑
- 预览数据集数据
- 删除数据集

数据集列表应展示：

- 数据集名称
- 所属人
- 创建时间
- SQL 逻辑摘要

数据集预览功能应支持查看部分样例数据，点击预览可以查看前100行数据，以表格形式展示。

### 5.6 数据连接管理

管理员可进入数据连接页面，维护外部数据库或数据服务连接。

页面应支持：

- 绑定新数据源
- 查看数据源列表
- 配置连接参数
- 删除连接
- 查看连接状态

数据源列表应展示：

- 数据库类型
- 连接名称
- 节点地址
- 连接账号
- 连接状态

当前界面体现的数据源类型包括：

- MySQL
- PostgreSQL
- SQL Server

连接状态包括：

- 连接成功
- 验证中 / 待验证
- 连接失败

### 5.7 用户管理

管理员可进入用户管理页面，维护系统账号、角色和权限。

页面应支持：

- 创建新用户
- 查看用户列表
- 选择单个或多个用户
- 批量删除用户
- 编辑用户
- 删除用户
- 分配权限

用户列表应展示：

- 用户名
- 邮箱
- 用户角色
- 激活状态
- 操作入口

用户状态包括：

- 已激活
- 待处理 / 待激活

用户角色至少包括：

- 系统管理员
- 管理员
- 普通用户


### 5.8 系统设置

管理员可进入系统设置页面，配置平台全局显示信息。

页面应支持：

- 修改系统名称
- 选择系统图标
- 上传自定义 Logo
- 移除自定义 Logo
- 预览侧边栏展示效果
- 保存配置

系统图标应提供预置图标选项。上传自定义 Logo 后，应优先展示 Logo，并清除当前图标选择。

Logo 上传要求从界面推断为：

- 支持 PNG、SVG、JPG
- 建议 1:1 比例

### 5.9 个人设置

用户可通过左下角用户菜单进入个人设置弹窗。

个人设置包含两个区域：

- 基本资料
- 安全设置

基本资料应支持：

- 查看当前头像
- 上传或更换头像
- 查看用户名
- 查看当前角色

头像上传要求从界面推断为：

- 推荐 200x200 分辨率
- 支持 JPG、PNG
- 最大 2MB

安全设置应支持修改登录密码，包括：

- 输入原密码
- 输入新密码
- 确认新密码
- 显示或隐藏新密码
- 提交修改
- 修改成功提示

密码强度：必须大于6位数，大小写、数字和字母；
过期策略：没有操作60min后过期；
错误提示：输入的账号或密码错误！

### 5.10 全局搜索与通知

顶部区域预留了全局搜索框和通知入口。

全局搜索可用于搜索平台内资源，建议搜索范围包括：

- 看板

当前搜索只能搜索到看板，点击搜索结果可以直接显示看板（不开启新的页面）


## 6. 权限要求

系统应根据用户角色控制菜单和页面访问权限。

- 系统管理员可访问全部模块。
- 管理员可访问数据集管理和我的看板
- 普通用户仅可访问“我的看板”和个人相关功能。
- 未登录用户不可访问工作台页面。
- 无权限用户访问受限页面时，应返回默认首页或展示无权限提示。

暂时不需要以下权限：

- 角色定义
- 权限点清单
- 看板级权限
- 数据集级权限
- 数据源级权限
- 用户操作审计要求

## 7. 关键流程

### 7.1 登录流程

1. 用户访问系统。
2. 系统判断用户是否已登录。
3. 未登录用户进入登录页。
4. 用户输入用户名和密码。
5. 系统校验凭证。
6. 校验成功后进入主工作台。
7. 校验失败时展示错误提示。

### 7.2 查看看板流程

1. 用户进入主工作台。
2. 用户展开“我的看板”。
3. 用户选择目标看板。
4. 系统加载看板配置和数据。
5. 用户调整日期范围或筛选条件。
6. 用户点击运行查询。
7. 系统刷新看板内容。

### 7.3 管理看板流程

1. 管理员进入看板管理。
2. 管理员搜索或浏览看板。
3. 管理员新建、编辑、删除或修改图标。
4. 系统保存变更。
5. 已发布看板在用户可访问范围内展示。

### 7.4 创建数据集流程

1. 管理员进入数据集管理。
2. 管理员点击创建数据集。
3. 管理员配置数据集名称、来源和 SQL / 查询逻辑。
4. 管理员预览数据。
5. 校验通过后保存数据集。
6. 数据集可被看板绑定使用。

### 7.5 配置数据源流程

1. 管理员进入数据连接。
2. 管理员点击绑定新数据源。
3. 管理员填写数据库类型、地址、账号和连接参数。
4. 系统验证连接可用性。
5. 验证通过后保存连接。

### 7.6 修改个人密码流程

1. 用户打开个人设置。
2. 用户进入安全设置。
3. 用户输入原密码、新密码和确认密码。
4. 系统校验密码规则。
5. 修改成功后展示成功提示。
6. 后续是否强制重新登录待补充。

## 9. 非功能需求

### 9.1 安全性

- 登录态应具备有效期控制。
- 密码应加密存储，不得明文保存。
- 数据源密码应加密保存或通过密钥管理服务托管。
- 管理操作应具备权限校验。
- 重要操作建议记录审计日志。

### 9.2 可用性

- 页面加载、保存、删除、查询等操作应有明确状态反馈。
- 表单提交失败时应展示可理解的错误信息。
- 侧边栏应支持收起，保证不同屏幕宽度下的可用性。

### 9.3 性能

- 看板查询应支持超时控制。
- 数据集预览应限制返回行数。
- 列表页应支持分页或虚拟滚动，避免大数据量卡顿。
- 全局搜索应避免阻塞主界面交互。

### 9.4 兼容性

- 支持主流现代浏览器。

## 10. 验收标准

- 未登录访问工作台时，系统应跳转登录页。
- 管理员登录后可看到全部管理菜单。
- 普通用户登录后只能看到允许访问的菜单。
- 用户可选择看板并进入看板展示页。
- 管理员可在看板管理中搜索看板并打开图标选择弹窗。
- 管理员可在数据集管理中搜索、编辑、预览和删除数据集入口。
- 管理员可在数据连接页面查看连接状态并进入参数配置入口。
- 管理员可在用户管理页面选择用户、批量删除、编辑和分配权限入口。
- 管理员可修改系统名称、选择图标或上传 Logo，并在预览区看到变化。
- 用户可打开个人设置，查看基础信息并提交密码修改。

## 11. 待补充问题

- 平台正式名称是否使用 AstroCore、TECH_CORE，还是另有名称？
- 登录认证方式是账号密码、单点登录，还是混合模式？
- 是否需要组织、部门、租户等多级权限模型？
- 看板编辑器是否由本系统提供，还是接入外部 BI / 可视化引擎？
- 数据集 SQL 是否允许用户自由编写，是否需要 SQL 安全沙箱？
- 数据源连接信息如何加密和托管？
- 看板查询是否需要缓存、定时刷新或导出能力？
- 通知中心的通知类型和触发规则是什么？
- 全局搜索的搜索范围、权限过滤和排序规则是什么？
- 是否需要操作审计、登录日志和数据访问日志？
